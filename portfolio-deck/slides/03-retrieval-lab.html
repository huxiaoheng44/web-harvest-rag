Research module

retrieval_lab: a standalone experimentation harness

Ten questions about retrieval quality, each answered with real numbers instead of assumptions — kept fully separate from the production app.

1

Chunking strategies

Fixed-size token windows vs. structural (heading-aware) splitting, compared head to head.

2

Vector vs. BM25 vs. Hybrid

FAISS cosine search, rank_bm25 keyword search, and two fusion strategies (linear + RRF).

3

Cross-encoder rerank

ms-marco-MiniLM reranking the hybrid candidate pool — tested, not assumed to help.

4

Chunk-size sweep

200 / 500 / 1,000-token chunks, measuring the completeness-vs-noise trade-off directly.

5

LLM query rewriting

Paraphrase each query into variants, retrieve for each, merge via reciprocal rank fusion.

6

Error-bucket analysis

Every imperfect-recall query auto-sorted into a failure category, then read through by hand.

7

Alpha tuning

Hybrid fusion weight swept on a held-out validation split, evaluated on a separate test split.

8

LLM-as-judge answer eval

Not just "was the right chunk retrieved" — is the generated answer actually faithful to it?
