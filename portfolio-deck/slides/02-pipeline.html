Architecture

Scrape → Chunk → Embed → Retrieve → Chat

Six stages, each independently swappable — the retrieval stage is where most of the engineering work in this project actually happened.

1

Scrape

config/sources.json → scraper.py pulls HTML pages and PDFs, extracts clean readable content.

→
2

Chunk

Fixed-size token windows or structural (heading-aware) splitting — compared systematically, not assumed.

→
3

Embed

OpenAI text-embedding-3-small, cached by content hash so re-runs don't re-pay the API.

→
4

Retrieve

Vector (FAISS / Supabase pgvector) + BM25, fused — the part covered in depth on the next slides.

→
5

Generate

FastAPI backend builds context from retrieved chunks, calls the chat model, stores the conversation.

→
6

Chat

Next.js UI — add sources, watch the index build live, ask questions, see cited sources.

Production uses Supabase pgvector for retrieval. A separate, standalone module — retrieval_lab — exists purely to answer "is this actually the best way to do stage 4?" without touching the live app. That's the rest of this deck.
